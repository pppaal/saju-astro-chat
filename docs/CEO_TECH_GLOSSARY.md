## 1. 기본 구조

### 🏪 카페로 이해하는 웹 서비스

| 용어         | 카페 비유                  | 실제 의미               | 예시                      |
| ------------ | -------------------------- | ----------------------- | ------------------------- |
| **Frontend** | 카페 홀 (고객이 보는 공간) | 웹사이트 화면, 디자인   | 버튼, 색상, 레이아웃      |
| **Backend**  | 주방 (뒤에서 일하는 곳)    | 서버, 데이터 처리       | 사주 계산, 결제 처리      |
| **Database** | 창고 (재료 보관)           | 데이터 저장소           | 고객 정보, 결제 내역      |
| **API**      | 주문 벨 (주방↔홀 연결)     | 데이터 주고받는 통로    | "사주 데이터 주세요" 요청 |
| **Cache**    | 미리 만든 샌드위치         | 자주 쓰는 데이터 빠르게 | 오늘의 운세 (12시간 저장) |
| **Server**   | 카페 건물 전체             | 서비스 돌아가는 컴퓨터  | Vercel, AWS 같은 곳       |
| **Client**   | 손님 (브라우저)            | 사용자 기기             | 크롬, 사파리              |
| **Request**  | 주문 ("카페라떼 주세요")   | 데이터 요청             | "사주 보기" 클릭          |
| **Response** | 커피 배달                  | 서버 응답               | 사주 결과 화면            |

### 데이터 구조

| 용어            | 창고 비유           | 실제 의미        | 예시                   |
| --------------- | ------------------- | ---------------- | ---------------------- |
| **Schema**      | 창고 설계도         | 데이터 구조 설계 | User, Post, Payment    |
| **Model**       | 물건 종류           | 데이터 타입      | User, SajuResult       |

| **Primary Key** | 고유 번호표         | 고유 식별자      | userId: 12345          |
| **Foreign Key** | 연결 번호표         | 다른 테이블 참조 | post.userId → user.id  |
| **Index**       | 창고 색인 (책 뒷면) | 빠른 검색용      | 이메일로 빠르게 찾기   |
| **Migration**   | 창고 리모델링       | DB 구조 변경     | "생년월일 컬럼 추가"   |
| **Seed**        | 초기 재고           | 테스트 데이터    | 더미 사용자 10명       |

## 7. 성능 & 최적화

### 속도 개선

| 용어               | 자동차 비유      | 실제 의미                | 목표             |
| ------------------ | ---------------- | ------------------------ | ---------------- |
| **Cache**          | 미리 준비된 부품 | 자주 쓰는 데이터 저장    | Redis 캐싱       |

| **PWA**       | 업그레이드 자전거 | 설치 가능한 웹앱 | 홈 화면에 추가              |

## 14. 에러 코드

### HTTP 상태 코드 (택배 상황)

| 코드    | 택배 비유               | 실제 의미        | 해결 방법        |
| ------- | ----------------------- | ---------------- | ---------------- |
| **200** | 배송 완료               | 성공             | -                |
| **201** | 주문 접수               | 생성 완료        | -                |
| **204** | 수령 확인 (영수증 없음) | 성공 (응답 없음) | -                |
| **400** | 주소 잘못 씀            | 잘못된 요청      | 입력값 확인      |
| **401** | 신분증 필요             | 인증 필요        | 로그인하세요     |
| **403** | 출입 금지               | 권한 없음        | 프리미엄 필요    |
| **404** | 주소 못 찾음            | 페이지 없음      | URL 확인         |
| **429** | 배송 한도 초과          | 요청 너무 많음   | 잠시 후 다시     |
| **500** | 물류센터 고장           | 서버 오류        | 개발자 수정 필요 |
| **502** | 중간 배송 실패          | 게이트웨이 오류  | 재시도           |
| **503** | 휴무                    | 서비스 불가      | 서버 점검 중     |

### 에러 처리

| 용어                     | 병원 비유 | 실제 의미      | 예시                     |
| ------------------------ | --------- | -------------- | ------------------------ |
| **Error**                | 병        | 오류           | "로그인 실패"            |
| **Exception**            | 응급 상황 | 예외 처리      | try-catch                |
| **Crash**                | 급사      | 앱 강제 종료   | 앱 튕김                  |
| **Bug**                  | 바이러스  | 프로그램 결함  | 버튼 안 눌림             |
| **Debug**                | 진단      | 오류 찾기      | console.log()            |
| **Stack Trace**          | 병력      | 에러 경로 추적 | "어디서 터졌나"          |
| **Fallback**             | 응급 처치 | 대체 방안      | OpenAI 안 되면 Replicate |
| **Graceful Degradation** | 양보 치료 | 기능 축소 운영 | Redis 없어도 작동        |

### 📞 질문 가이드

**"이거 왜 안 돼요?"**

1. 서버 상태 확인 (Uptime)
2. 에러 로그 확인 (Sentry)
3. 최근 배포 확인 (Deploy 이력)
4. 필요하면 Rollback

**"비용이 왜 올랐어요?"**

1. AI 사용량 확인 (Token)
2. 사용자 증가 확인 (MAU)
3. 캐시 활용 확인 (Cache Hit Rate)
4. 모델 최적화 (GPT-4o → mini)

**"언제 완성돼요?"**

1. Sprint 확인 (2주 단위)
2. Backlog 확인 (대기 작업)
3. P0/P1 우선순위 확인
4. Blocker 있나 확인

**"이거 고치려면 얼마나 걸려요?"**

- P0 (긴급): 즉시 ~ 당일
- P1 (중요): 1-3일
- P2 (보통): 1-2주
- P3 (낮음): 다음 Sprint
