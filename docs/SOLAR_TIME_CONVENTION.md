# Solar Time Convention — 시각 보정 정책

## 결론 (요약)

본 엔진은 **출생 시각을 그대로 표준시(KST = UTC+9)로 해석**하고, 진태양시(眞太陽時) 보정을 *적용하지 않습니다*. 사용자가 입력한 `06:40 KST`는 사주 계산에서도 점성 계산에서도 모두 `06:40 KST`로 사용됩니다.

이는 **의식적인 설계 결정**이며 버그가 아닙니다. 다만 일부 전통 사주 학파는 다른 보정을 적용하므로, 사용처에 따라 차이가 발생할 수 있음을 알려드립니다.

---

## 배경 — 시각의 세 종류

| 종류 | 정의 | 서울 기준 06:40 |
|---|---|---|
| **표준시** (KST) | 한국 표준시 (UTC+9, 표준 자오선 = 135°E) | 06:40 (사용자 입력 그대로) |
| **평균태양시** | 출생지 경도 기준 평균 태양시 | 약 06:08 (서울 126.98°E vs 135°E → -32분) |
| **진태양시** | 출생지 + 그 날의 태양 적도 위치 보정 (균시차 포함) | 06:08 ± 16분 (계절별 변동) |

서울은 표준 자오선(135°E)에서 약 8°02′ 서쪽이라, 평균태양시는 표준시보다 **약 32분 빠름**. 진태양시는 여기에 균시차(-16~+16분)를 더 적용.

---

## 본 엔진의 정책

### 사주
- 출생 시각을 표준시(KST)로 그대로 사용
- 시지(時支) 결정 시 **표준시 06:40 → 卯時** (5:30~7:30)
- 일주(日主) 변경 시점도 표준시 자정 기준
- 절기(節氣) 시각은 한국천문연(KASI) 데이터 그대로 사용 (KST 자체)

### 점성
- Swiss Ephemeris를 통해 계산. 입력 시각을 KST로 받아 UTC로 변환 후 ephemeris 호출
- 출생지 위경도(`latitude`, `longitude`)는 별도 입력 — house cusps와 ASC/MC 계산에 사용
- 점성에서는 별도의 평균태양시·진태양시 보정 없음 (각 천체의 정확한 천문학적 위치만 사용)

---

## 학파별 차이

### 표준시 사용 (본 엔진)
- 현대적·실용적 접근
- 사용자가 입력한 시각이 그대로 결과에 반영 — 일관성·재현성 좋음
- 한국 전통 만세력 다수가 이 방식 채택

### 평균태양시 보정
- 일부 자평진전·강호 학파가 채택
- 서울 출생자에게 **약 -32분 보정** 적용 (06:40 → 06:08)
- 시지가 卯時 경계(06:00)에 가까운 경우 시지 자체가 바뀔 수 있음 → 시주 다름

### 진태양시 보정
- 가장 정밀한 전통적 방식
- 평균태양시에 균시차(equation of time) 더 적용
- 입동~동지: -16분, 입하~하지: +14분 등 계절별 변동
- 일부 적천수 학파가 채택

## 영향 — 시지 경계 케이스

서울 출생, 표준시 기준 시각 별 시지 변환:

| 표준시 | 평균태양시 | 시지 (표준시 기준) | 시지 (평균태양시 기준) | 차이 |
|---|---|---|---|---|
| 23:30 | 22:58 | 子時 | 亥時 | **시주 다름** ⚠️ |
| 06:40 | 06:08 | 卯時 | 卯時 | 동일 |
| 11:30 | 10:58 | 午時 | 巳時 | **시주 다름** ⚠️ |

→ 표준시가 시간 경계(예: 1:30·3:30·5:30·7:30·9:30·11:30·13:30·15:30·17:30·19:30·21:30·23:30) 부근일 때 보정 여부가 시주를 바꿈.

## 사용자 가이드

본 엔진의 결과는 **표준시 기준**입니다. 만약:

- 다른 만세력(특히 평균태양시 사용)과 결과가 다르면 → 시지 보정 차이 가능성
- 시간 경계 부근(±15분) 출생자 → 보정에 따라 시주 바뀔 수 있음, 두 가지 결과 모두 검토 권장
- 진태양시 정밀 보정 필요 시 → 별도 모듈로 추가 가능 (현재 미구현)

## 향후 계획 (옵션)

진태양시 보정이 필요한 사용자를 위해:

```ts
interface BirthInput {
  // ... existing fields
  solarTimeMode?: 'standard' | 'meanSolar' | 'trueSolar'
}
```

- `'standard'` (기본): 현 동작
- `'meanSolar'`: 출생지 경도로 평균태양시 보정
- `'trueSolar'`: 평균태양시 + 균시차 (계절 변동 포함)

각 모드의 결과 차이를 사용자에게 보여주는 UI도 가능. 현재는 기본 모드만 지원.
