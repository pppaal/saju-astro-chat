# Solar Time Convention — 시각 보정 정책

## 결론 (요약)

본 엔진은 **사주 계산에서 평균태양시(平均太陽時) 보정을 적용**합니다. 출생지(또는 위치) 경도가 주어지면, 시계 시각을 그 지역의 *해 기준 시각*으로 옮긴 뒤 네 기둥(년·월·일·시)을 산출합니다. **점성 계산에는 보정이 없습니다**(천문학적 실제 위치를 그대로 사용).

보정 공식의 **단일 진실원천(SSOT)** 은 [`solarTimeCorrectionMinutes()`](../src/lib/saju/timezone.ts) 입니다. 사주의 모든 시각 보정은 이 함수 하나만 호출합니다.

```
보정분 = round((경도 − 표준자오선) × 4)
표준자오선 = 그 시점 타임존 UTC offset(분) / 60 × 15   (KST=135°, 베이징=120°, EST=−75° …)
평균태양시 = 시계 시각 + 보정분
```

> ⚠️ 과거 이 문서는 "표준시 그대로, 보정 미적용"이라고 적혀 있었으나, 코드는 이미 평균태양시 보정을 적용하고 있었습니다. 이 문서는 2026-06-06 코드 실제 동작에 맞춰 다시 작성되었습니다.

## 배경 — 시각의 세 종류

| 종류             | 정의                                         | 서울 06:40          | 본 엔진     |
| ---------------- | -------------------------------------------- | ------------------- | ----------- |
| **표준시** (KST) | 한국 표준시 (UTC+9, 표준 자오선 135°E)       | 06:40 (입력 그대로) |             |
| **평균태양시**   | 출생지 경도 기준 평균 태양시 (경도 보정)     | 06:08 (−32분)       | ✅ **사용** |
| **진태양시**     | 평균태양시 + 균시차(equation of time, ±16분) | 06:08 ± 16분        | ❌ 미적용   |

서울은 표준 자오선(135°E)에서 약 8°02′ 서쪽 → 평균태양시는 표준시보다 약 32분 빠름. **균시차(계절 ±16분)는 적용하지 않습니다** — 천안/부산 같은 경도 차이를 잡는 게 목적이고, 균시차는 별개의 정밀도 옵션이기 때문입니다.

### 도시별 보정 예시

| 도시                            | 경도     | 보정분 | 진태양 정오(시계) |
| ------------------------------- | -------- | ------ | ----------------- |
| 천안                            | 127.15°E | −31분  | 12:31             |
| 부산                            | 129.08°E | −24분  | 12:24             |
| 서울                            | 126.98°E | −32분  | 12:32             |
| 카슈가르(중국 서부, 베이징시간) | 76°E     | −176분 | 14:56             |
| 뉴욕(4월, EDT)                  | −74°W    | −56분  | —                 |

> 중국 서부처럼 타임존이 지리적으로 넓은 지역은 보정이 1~3시간에 달해, 보정 없이는 시진(時辰)이 통째로 어긋납니다. 그래서 전 세계 일관 정확도를 위해 경도 보정이 필요합니다.

## 적용 범위 (RAW 계산 기준)

| RAW 요소                                  | 시각 기준                                              | 위치                          |
| ----------------------------------------- | ------------------------------------------------------ | ----------------------------- |
| **본명 사주 — 년·월·일·시 네 기둥**       | 평균태양시 (보정된 instant 하나에서 전부 산출)         | `saju.ts` `effectiveDateTime` |
| **달력 시진 (時)** — forecast 시각별 길흉 | 평균태양시 (시진 경계를 보정)                          | `saju-hour.ts`                |
| **달력 일진 (日)** — 그 날짜의 일주       | 캘린더 날짜 기준 (날짜 = 그 날의 일진, 보정 대상 아님) | `computeDayStem`              |
| **점성** (본명·트랜짓)                    | 실제 UTC 천문 위치 (보정 개념 없음)                    | Swiss Ephemeris               |

> **본명 보정의 핵심:** 보정은 *시(時)만*이 아니라 **출생 instant 자체**를 옮긴다. 그래서 자정·입춘·절기 경계 출생자는 일주·월주·년주까지 올바르게 이동한다. (예전엔 시주 lookup 에만 보정이 들어가 경계에서 불일치가 있었음 — 2026-06-06 수정.)

> **일진을 보정하지 않는 이유:** 본명은 *태어난 순간*이라 보정이 필요하지만, 달력의 "오늘의 일진"은 _날짜의 속성_(예: 甲子일)이라 날짜마다 고정값이다.

## 학파별 차이 (참고)

| 방식                       | 채택               | 본 엔진        |
| -------------------------- | ------------------ | -------------- |
| 표준시 그대로              | 현대 만세력 다수   |                |
| **평균태양시 (경도 보정)** | 자평진전·강호 일부 | ✅             |

## SSOT 계약

- 사주의 모든 시각 보정은 **`solarTimeCorrectionMinutes(instant, longitude, timeZone)` 하나만** 호출한다. 인라인으로 `(경도−자오선)×4` 를 복붙하지 말 것.
- 회귀: [`tests/lib/Saju/solarTimeCorrection.test.ts`](../tests/lib/Saju/solarTimeCorrection.test.ts) 가 도시별 보정값을 고정.
- `longitude` 미상 시 보정 0 → 옛 동작(캐시 호환) 보존.
