# 캘린더·리포트 해석 정리 계획

> 두 SSOT 와 정합: `운흐름.md`(캘린더) · `REPORT_UX_PLAN.md`(리포트).
> (작성 2026-06-07)

## 0. 한 줄 프레임 — 명사 vs 동사

|                | 리포트 (destinymap)                           | 캘린더 (운흐름)                             |
| -------------- | --------------------------------------------- | ------------------------------------------- |
| 정체           | **그 사람 고유의 에너지** (원국·고정)         | **시간 위의 흐름** (대운→세운→월운→일진)    |
| 사주/점성 성격 | **명사** — 격국·일주·십신분포·dignity·Almuten | **동사** — 형충·대운전환·트랜짓·교차활성    |
| 근거           | REPORT_UX_PLAN §6D (12 교차 카드, 테마카드 X) | 운흐름 §0.5.8 (명사는 "흐름과 무관" → 제외) |

→ **같은 계산(saju/astro)**, **같은 교차 지식**을 쓰되, 리포트는 _고정 에너지에_,
캘린더는 _시간 발동에_ 적용한다. 둘은 합치는 게 아니라 **지식은 공유 / 적용은 분리**.

## 1. 공유 (둘 다 끌어쓰는 한 벌)

- **계산 SSOT:** `lib/saju` · `lib/astrology` (원국 차트). — 변경 없음.
- **교차 지식 SSOT:** `lib/calendar-engine/data/saju-astro-mapping.ts` (사주요소×점성요소 → meaning)
  - `lib/saju/elementBridge.ts`. 리포트·캘린더가 _같은 사전_ 참조.
- **요소 해석 SSOT:** `lib/chart-dictionary/*.json` (명사 per-요소 풍부 해석, ko/en).

## 6. 기존 해석 파일 중복 정리 (새로 만들지 말고 한 벌로)

> "옛날 해석이 여러 세대로 쌓여 같은 걸 3벌씩" 정리. **제일 풍부한 걸 SSOT 로 남기고,
> 소비처를 거기로 돌린 뒤 나머지 삭제.** (소비처 타입체크로 검증)

### 6.1 일주(60갑자) 해석 — 3벌 → 1벌 ⭐ (가장 명확)

| 파일                              | 줄  | 성격                                                                    | 소비 | 처리                                                  |
| --------------------------------- | --- | ----------------------------------------------------------------------- | ---- | ----------------------------------------------------- |
| `chart-dictionary/ilju-60.json`   | 962 | character/strength/weakness/career/love, **ko+en, 대화체(최신·최풍부)** | 1    | ✅ **SSOT 유지**                                      |
| `saju/iljuDictionary.ts`          | 773 | archetype ko+en                                                         | 3    | 소비처 → ilju-60 으로, 후 삭제                        |
| `saju/pillar-lookup/ilju-data.ts` | 561 | personality/career(한글, 옛)                                            | 2    | ⚠️ calc 필드(IljuInfo) 섞였는지 확인 후 해석부만 정리 |

### 6.2 십신·오행·십이운성·신살 — 2벌 (입도 다름 → 신중)

| 파일                                          | 내용                               | 소비                                        |
| --------------------------------------------- | ---------------------------------- | ------------------------------------------- |
| `saju/data/interpretations.json` (1156)       | per-**십신**/오행/12운성/신살 상세 | api/saju, saju/index, calendar lifetimeFlow |
| `chart-dictionary/{sibsin-category,...}.json` | per-**카테고리**(비겁/식상…)       | report, sibsinAnalysis                      |

→ 순수 중복 아님(십신 단위 vs 카테고리 단위). **1차: 같은 단위 중복 키만 한쪽으로**, 단위가 다른 건 역할 주석만 명확히.

### 6.3 교차 — 합치지 말 것 (지식만 공유)

- `data/saju-astro-mapping.ts` = 공유 사전(유지). `report/natalCross.ts`(명사 적용)·
  `cross-activation.ts`(동사 적용)는 같은 사전을 쓰는 _두 적용_ — 유지.

### 6.4 순서

1. **일주 3→1** (6.1) — 소비처 6곳 ilju-60 으로 이전, 2파일 삭제, 타입체크. (가장 깨끗)
2. 십신/오행 키 정리 (6.2) — 단위 확인 후.
3. (테마 제거 후) humanSemantics\* 잔재 점검.

> 1·2 는 여기서 안전하게(테스트). 3 은 출력·UI 바뀌니 골든 재생성 + 로컬 시각검증.
