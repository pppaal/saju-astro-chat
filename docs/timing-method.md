# 타이밍 엔진 — 우리 방법 + 과학적 근거

> 사주(동양)와 점성(서양)을 교차해 **시간 단위부터 인생 전체까지 모든 타이밍**을 내는 방법.
> **파이프라인 건전성 검증 완료**(랜덤 30개 사주 × 4스케일 크래시 0·분포 정상) — 단 **예측 타당도는 미검증**(정답 데이터 없음, §5). 흩어진 프로토타입/커밋을 하나로 정리.
> 브랜치: `claude/gyeol-to-heureum-bQpP2` · 프로토타입: `scripts/flow-proto.ts`(git 이력에 v1~v14)
> **상태: 적대 검증(3-Opus) 반영 — 핵심 방법을 fusion/convergence 분리 + 기하평균 + copula로 교정. 일부는 권장(미프로토타입).**

---

## 0. 한 줄 요약

> **사주 점수와 점성 점수를 각각 따로 낸다 → 둘이 같은 방향으로 강하게 만나는 시기가 "타이밍" → 그때 평소보다 유난히 튄 분야가 "무슨 타이밍인지"다. 시간·일·월·인생 전부 같은 방식, 배율만 다르다.**

---

## 1. 재료 — "신호(signal)"

사주·점성이 각각 작은 운 조각(신호)을 만든다. 신호 하나 = 5개 정보:

| 정보 | 뜻 | 예 |
|---|---|---|
| polarity | 좋다(+)/나쁘다(−), −3~+3 | 토성 스퀘어 −2 |
| weight | 얼마나 센지(정확도·중요도) 0~1 | orb 타이트하면 ↑ |
| 시간창 | 시작–정점–끝 | 5/8 시작, 5/11 정점, 5/14 끝 |
| 테마 | 분야 | love/money/career/health/growth |
| layer | 영향 길이(스케일) | 10년/1년/1달/하루/시간 |

- **점성 신호**: 트랜짓·식·프로그레션·리턴·프로펙션·ZR·라이프사이클·행성시·소행성·미드포인트·항성 등
- **사주 신호**: 대운·세운·월운·일진·시주·신살·형충회합·십신·용신

→ Layer 0(천문·사주 사실 계산)은 이미 정확(테스트 통과). 신호 추출기 = **모든 상위 방법의 입력 파이프라인**.

## 2. 방법 — 5단계

### 단계 1. 점수를 **둘 따로** 매긴다
그 시점에 작동하는 신호를 모아 → **사주 점수** 1개 + **점성 점수** 1개. 따로 낸다.
- 이유: 사주·점성은 **다른 두 전통(두 점쟁이)**. 따로 내야 "둘이 일치하나"가 의미를 가진다.
- 모든 화면 위젯이 **같은 이 두 숫자**만 보게 한다 → "축은 aligned인데 교차 28%" 같은 모순이 구조적으로 불가능.

### 단계 2. 점성 "홍수"를 솎는다
점성은 하루 150+개 신호(소행성·미드포인트·행성시 등)가 쏟아져, 다 평균 내면 **변별이 사라져 축이 죽는다**(실측: 변동 sd 0.031 = 일직선).
- **해법**: 레이어별로 (weight×중요도) **상위 N개만** 점수에 반영(적응형 N, 보통 5~8). → 축이 살아남(sd 0.08~0.2).
- **버리는 게 아니다**: 솎인 신호는 **상세·테마·설명에 그대로** 남는다(점수에만 안 들어감).
- ⚠️ 이 "top-N"은 검증된 실용책이나 임의 컷. §5에 더 정식한 대안.

### 단계 3. 0~100으로 환산 — **고정 기준**으로
각 점수를 "그 사람 평소" 기준으로 0~100 환산(50=보통).
- **고정 기준 창**(예: 그 해 ±몇 달)에서 **신호도 보정값도 1회 계산**, 모든 뷰는 거기서 **잘라 쓰기**.
- 이유: 안 그러면 같은 날이 보는 창(주/달)에 따라 점수가 달라진다(실측: 1주 창에서 opposed 5배 튐). 신호 자체가 빌드 창 경계서 잘려서(점성 연속 트랜짓) → **반드시 1회 빌드 후 슬라이스, 재빌드 금지**.

### 단계 4. 합치고 + **타이밍** 찾기 — ⚠️ **두 개의 다른 문제**
적대 검증 핵심: **점수 합치기(fusion)와 시기 일치(convergence)는 다른 문제 = 다른 도구.**

**(a) 종합점수 = fusion — 기하평균/MPI (산술평균 금지)**
- 산술평균은 (0,10)도 (5,5)도 똑같이 5 → **엇갈림을 평범으로 뭉개** 타이밍 신호를 스스로 파괴.
- **기하평균 / Mazziotta-Pareto Index(불균형 패널티)** = 한쪽이 낮으면 종합이 끌려내려감(비보상). 필수.
- (정직: 기하평균은 보상성을 *줄일* 뿐 두 축 *상호작용*을 모델링하진 못함 → (b) copula가 그걸 함.)

**(b) 타이밍 = convergence — copula 꼬리의존(λ), coherence ✗**
- "두 시스템이 **같은 극단으로 동시에** 튀는 드문 시기"가 타이밍의 정의.
- **copula 상·하 꼬리의존(λ)**이 정확히 이걸 잰다: Gumbel=상측(둘 다 길 ★) / Clayton=하측(둘 다 흉 ▼) → **길/흉 비대칭**.
- ❌ **coherence(MSC) 폐기**: 제곱이라 부호 무관 → "둘 다 높음"과 "한쪽만 높음"을 **구분 못 함**(우리 정의와 정반대). 짧고 비정상인 데이터에도 부적합.
- 대안: Bayesian/Dempster-Shafer 융합(소스 간 conflict 정량화 = "엇갈림" 정식 버전)도 평균보다 우리 의도에 가까움.
- 강한 수렴은 드물어야 의미 있다(한 달에 0~수일).

### 단계 5. "무슨 분야" 붙이기 — 기준선 이탈
- 5개 테마(연애·돈·일·건강·성장) 각각 **그 사람 평소(기준선) 대비 유난히 튄 정도**를 본다.
- 유난히 튄 테마 있음 → "그 분야 타이밍"(예: 연애 기회).
- 아무 테마도 안 튐 → **억지 라벨 금지, 그날 핵심 신호로 설명**("토성·해왕성 압박").
- 이유: 날것으로 보면 'career' 태그가 과다 부착돼 매일 "일"로 도배됨. 기준선 이탈로 봐야 진짜 분야가 갈린다.

## 3. 모든 스케일 — 같은 방식, 배율만 다름

신호의 layer(영향 길이)로 줌이 된다. 큰 줌 = 큰 신호만, 작은 줌 = 잔 신호까지.

| 보기 | 사주 | 점성 | 길흉 산출 | 분야 |
|---|---|---|---|---|
| **인생**(10년~) | 대운 | 토성회귀·목성회귀·천왕성반대·카이런회귀 | 대운이 **용신**에 맞나 | **십신**(재성=재물·연애기, 관성=직업기, 인성=성장기…) |
| **연/월/일** | 세운·월운·일진 | 트랜짓·리턴·프로그레션 | 두 축 수렴 | 테마 기준선 이탈 |
| **시간**(時) | 시주(12시진) | 행성시(칼데안) | 두 축 수렴 | — |

- **인생 길흉**: 사주 대운(용신)이 주연. 점성은 **드문 일생 마커**(토성회귀 등)만 — ⚠️ 이 스케일은 점성 기여가 거의 0이라 **사실상 단일 축**(전 스케일 "두 축 통일" 주장이 인생에선 깨짐 — 정직 표기).
- 예(1993생): 유년 시련(기신 토) → 20대 성장기(인성/화) → 40·50대 직업기(관성/목).

**⚠️ 도메인 정확성 한계 (적대 검증 — 인생 스케일이 가장 약함):**
- **대운=용신 오행 단순매칭**은 근사. 통근(通根)·조후(調候)·합충(合沖)·개두절각을 무시 → **길흉이 반대로 날 수 있음**. 천간/지지 분리 평가도 안 함.
- **십신→5분야 매핑이 성별 무시 = 버그.** 연애·배우자성은 **남명=재성, 여명=관성**. "재성=연애" 고정은 여성 차트 연애 타이밍을 통째로 어긋나게 함 → **성별 분기 필수.** 식상·비겁 매핑도 누락.
- 십신 분야의 길흉은 **그게 용신이냐 기신이냐에 종속**(기신 재성 = 재물흉) → 분야×용신을 곱으로 결합해야.
- **행성시 sect 무시 = 버그.** 토성=주간 완화/화성=야간 완화인데 flat 처리 → 흉성 절반 오판. (행성시가 일출·일몰을 쓰면서 정작 주야 길흉은 무시 = 모순)
- 카이런/천왕성반대 = **현대 점성**(헬레니즘 아님) 혼입 — 전통 출처 명시 필요.
- 시주(2h)·일진(하루)은 **이산(box)**, 점성 트랜짓은 **연속(curve)** → 같은 곡선 로직 씌우면 사주에 **가짜 정점** 발명. 신호 형태 타입 분리 필요.

## 4. 과학적 근거 — 우리 방법은 확립된 분야의 적용

우리가 끙끙대며 만든 것들이 **이미 학계가 푼 문제**다. 정식 명칭과 대응:

**구조적 통찰(적대 검증 1순위): 두 개의 다른 문제를 섞지 말 것.**
- ⓐ **점수 합치기(fusion)** — 한 시점 두 점수 → 종합. 도구: 합성지표 비보상 집계 / 점수퓨전 / Bayesian-DS.
- ⓑ **시기 일치(convergence)** — 시간축에서 둘이 같이 극단. 도구: **copula 꼬리의존**.
- 둘은 다른 도구다. coherence/DTW/칼만은 *긴·정상* 시계열용이라 우리 *짧고 비정상·사건성* 데이터엔 과함.

| 우리가 한 것 | 확립된 방법 | 채택 여부 |
|---|---|---|
| 두 점쟁이 점수 합치기 | 합성지표(OECD) / 점수퓨전(생체인식) / Bayesian-DS | ✅ |
| "좋은+나쁜=평범" 막기 | **기하평균 / MPI(불균형 패널티)** = 비보상 집계 | ✅ **필수**(산술평균 금지) |
| 두 축 상호작용 모델링 | copula / 명시적 곱항(tension) | ✅ (기하평균은 *완화*만, 모델링 아님) |
| **타이밍=둘 다 극단 동시** | **copula 상·하 꼬리의존 λ** (Gumbel=길/Clayton=흉) | ✅ **권장**(우리 정의와 정확 정합) |
| ~~수렴=coherence(MSC)~~ | ~~Magnitude-Squared Coherence~~ | ❌ **폐기**(부호 무관→엇갈림 구분 못함, 짧은계열 부적합) |
| 보정(강건) | **MAD/percentile-rank** 정규화 + scale 상한 | ✅ (tanh도 12/sd 폭발은 못 고침) |

### 권장 업그레이드 (과학적, 미프로토타입)
1. **fusion: 산술평균 → 기하평균/MPI** — 필수·저비용. 상쇄 제거.
2. **convergence: coherence 폐기 → copula 꼬리의존(λ)** — "둘 다 극단" 직격, 길/흉 비대칭.
3. **정규화: 12/sd 폭발 → MAD/percentile + 상한** — 작은 sd에서 노이즈 증폭 차단.
4. **점성 솎기: top-N → 종류별(per-kind) tier 집계** — 임의 N 없음(top-N의 "사주 sd 맞추기"는 아티팩트).

> 정정: 이전 "진짜 레버 = 정규화+비보상집계"는 절반만 맞다. 진짜 미싱 피스는 **fusion(기하평균/DS)과 convergence(copula λ)를 분리**해 각각 맞는 도구를 쓰는 것. "tension=기하평균"도 과장이었다(완화≠모델링).

## 5. 검증 — 무엇을 보장하고 무엇을 못 하나 (정직)

- **랜덤 30개 사주 × 4스케일 → 크래시 0, 분포 정상**.
- ⚠️ **이건 "파이프라인 건전성(smoke test + 분포 위생)"이지 "예측 타당도"가 아니다.** 합격기준(축 sd≥0.03, opposed, ★↔용신)이 **전부 내부에서 만든 숫자로 자기를 채점 = 순환논법**. 난수를 넣어도 "축 살아있음"은 통과. **운이 실제로 그 방향이었는지는 0% 보장 못 함**(정답 데이터 없음).
- ✅ 보장: 30입력 예외 0, 출력이 degenerate(일직선/전부동일) 아님, 두 축이 과도하게 반대로만 가진 않음.
- ❌ 미보장: 예측 적중, ★/▼의 실제 의미, 테마 라벨 정확도, N·tier·임계의 일반화.
- 진짜 타당도 검증은 **무작위 보류·맹검 A/B + 사용자 피드백**(§6)으로만 가능.
- 적대 검사로 잡은 결함: 점성축 고착 · 짧은창 불안정 · 인생 점성홍수 · 합격기준 오류 / + 3-Opus 리뷰로 coherence 오선택·평균 상쇄·sect·연애성별·12/sd 폭발 추가 발견.

## 6. 알려진 이슈 / 로드맵

**Layer 0 버그 2개 (별도 수정)**
- 🔴 극지(>66.5°) 하우스 계산 크래시 — Placidus 미정의, fallback(Whole Sign) 없음.
- 🔴 용신 데이터 모순 — 한 사주에서 같은 오행이 용신·기신 동시 표기.

**도메인 버그 (적대 검증 — 수정 필요)**
- 🔴 행성시 sect 미적용(토성·화성 길흉 절반 오판).
- 🔴 연애 테마 성별 미분기(여명=관성인데 재성 고정).
- 🟠 대운=용신 단순매칭(통근·조후·합충·천간지지분리 무시).
- 🟠 십신 분야 길흉이 용신/기신에 종속돼야(곱).

**미해결 결정**
- fusion 집계: 산술평균(현재) → **기하평균/MPI** 전환(필수) — 프로토타입 후 확정.
- convergence: **copula 꼬리의존** 도입(coherence 폐기).
- 점성 솎기: top-N → 종류별 tier 집계.
- 정규화: 12/sd → MAD/percentile + 상한.
- 영역 언어 통일: 하루=5테마 / 인생=십신.

**ML 로드맵 함정 (정직)**
- 피드백 회귀의 라벨 오염: **자기실현예언**(앱이 "길"이라 하면 행동 바뀜), **생존·확증편향**(응답자 비대표+Barnum 해석), **누설**(피처=규칙이라 규칙 재학습). → **무작위 보류·맹검 A/B**로만 인과 타당도 측정.
- "도메인 선형" 가정은 부정확(형충회합·어스펙트 결합은 비선형) → "정규화 선형(릿지/라쏘) + 명시적 상호작용항"이 정확.

**가중치 학습 로드맵 (ML은 2단계)**
- ML(회귀·딥러닝)은 **정답 데이터(y)가 없어 지금 불가** — "운이 좋았다"는 측정된 적 없음.
- 경로: ①규칙 기반(이 문서) 출시 → ②사용자 피드백(👍/👎) 수집 → ③**회귀로 신호 가중치 학습**(손튜닝 tier/N 대체) → ④규칙(설명) + 학습가중치(정확도) 블렌드.
- **회귀 > 딥러닝**: 도메인이 대체로 선형 + 데이터 적어도 됨 + **"왜?" 설명 가능**(운세 가치의 절반).
- 규칙 기반 extractor는 어느 경우든 **ML의 피처 파이프라인**으로 재사용.
