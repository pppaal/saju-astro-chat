# 통합 리포트 UX 계획 (사용자 친화 + 정통 깊이)

> 통합 리포트 (차트 페이지) 가 *정통 점성·사주 깊이* (Almuten / Arabic Lots /
> dignity 5-tier / 격국 / 용신 등) 를 노출하되 일반 사용자가 부담 없이 이해
> 할 수 있게 정보 계층화 + 친화 라벨 + 운명상담사 LLM 연계 설계.
>
> v1 (2026-06-06): 초안. §6.G `buildReportContext` 새 엔진 implementation
> 전 UX 결정 SSOT.

## 1. 문제

```
정통 데이터가 풍부할수록 → 일반 사용자에게 외계어 부담
  · "Almuten Figuris 가 뭐야?"
  · "용신은 木인데 喜는 火, 忌는 水, 仇는 금? 무슨 소리?"
  · "dignity 5-tier 라는데 5단계가 뭐임"

vs 단순화 → 마니아 만족 0
  · "어차피 가짜 score 시스템 (옛 destiny-matrix) 거부함"
```

**목표**: 초보 한눈에 이해 + 마니아 정통 깊이 *동시에*.

## 2. 사용자 3 페르소나

| 페르소나 | 비율 (가정) | 기대 | 부담 |
|---|---|---|---|
| **초보** | ~70% | "내 사주 어때?" 한 줄 답 | 한자 / 영문 용어 / 표 다수 |
| **일반** | ~25% | 격국·용신·12운성 정도는 이해 | Lots / Almuten / dignity 5-tier 모름 |
| **마니아** | ~5% | Hellenistic 정통 깊이 + 점수 | 단순화 / 축약 싫음 |

## 3. 정보 계층 — 3 단계 (Progressive Disclosure)

```
┌─────────────────────────────────────────┐
│ Level 1 — 한눈에 (초보용, default 펼침)  │
│                                           │
│ 12 교차 카드 (정체성/욕망/사회역할 등)    │
│ 차트 데이터들이 자동 합성된 추상 결론      │
│ ← 누구나 한국어로 이해 가능                │
├─────────────────────────────────────────┤
│ Level 2 — 차트 데이터 (일반용, default 펼침)│
│                                           │
│ ▸ 사주 4기둥 표 + 신살 + 대운 흐름          │
│ ▸ 점성 행성 표 + 하우스 + aspects         │
│ ← 친화 라벨 + 툴팁 (hover/tap 시 한 줄 설명) │
├─────────────────────────────────────────┤
│ Level 3 — 정통 깊이 (마니아용, default 접힘)│
│ [▶ 정통 점성 깊이 보기] 클릭해서 펼침       │
│                                           │
│ ▸ 7 Arabic Lots (Fortune/Spirit/Eros/...) │
│ ▸ Almuten Figuris (주재 행성 + 점수 분해)  │
│ ▸ dignity 5-tier 표 (planet × 5단계)      │
│ ▸ 본명 aspects minor 추가                 │
│ ← 영문 + 한국어 같이, 정밀 데이터 그대로    │
└─────────────────────────────────────────┘
```

**원칙**:
- 초보가 Level 3 안 펼치면 외계어 0
- 마니아는 한 번 펼치면 정통 깊이
- Level 1 (교차 카드) 만으로도 충분히 의미 있는 한 페이지

## 4. UI 패턴

### A. 친화 라벨 + 툴팁
- 한글 친화 라벨이 메인, 영문 / 한자 부제로
- 어려운 용어 위에 `?` 아이콘 → hover/tap 시 한 줄 설명
- 모바일: tap 으로 popover

### B. Level 3 접고 펼침 (Disclosure)
- 기본 접힘 (collapsed)
- 헤더 클릭 → 펼침 + smooth animation
- 사용자 선호 localStorage 저장 (한 번 펼치면 다음 방문 시 기억)

### C. 운명상담사 CTA — 차트 곳곳
- 각 섹션 옆에 작은 `💬 더 알고 싶으면` 버튼
- 클릭 → 운명상담사 페이지로 이동 + 자동 질문 prefill
  - 예: Almuten 섹션 → "내 주재 행성 의미를 자세히 알려줘"
  - 예: 격국 섹션 → "내 격국 정관격이 인생에 어떻게 작용해?"

### D. 정보 밀도 조절
- 데스크톱: 한 화면 풍부 (4 컬럼 grid)
- 모바일: 세로 스택 + 섹션 카드 간 간격 ↑

## 5. 용어 친화 매핑

### 사주 (이미 한글 — 친화도 OK)
| 정통 용어 | 친화 라벨 (메인) | 한 줄 설명 (툴팁) |
|---|---|---|
| 격국 | **사주 구조** (격국) | 사주의 본질 짜임새 — 정관격/식신격 등 |
| 용신 | **필요한 기운** (용신) | 사주 균형을 위해 필요한 오행 |
| 12운성 | **운명 12 단계** | 각 기둥이 거치는 단계 (장생/목욕/관대/...) |
| 신살 | **사주 마커** (신살) | 길흉을 표시하는 신호 (도화/공망/천을귀인) |
| 통근 | **일간 뿌리** | 일간이 지지에 뿌리 박혀 강한지 |
| 득령 | **계절 기운** | 일간이 그 계절의 기운을 받았는지 |
| 조후용신 | **계절 균형** | 한겨울/한여름 출생 → 보충 필요한 기운 |
| 십신 분포 | **에너지 분포** | 식상·재성·관성·인성·비겁 카운트 |
| 공망 | **빈 자리** (공망) | 일주 기준 비어있는 2 지지 |

## 6. 운명상담사 LLM 연계 (핵심 CTA)

차트 페이지가 *데이터 전시* 라면, **해석은 운명상담사 LLM** 의 일.
차트 데이터를 자연스럽게 운명상담사로 흘려보내는 UX:

### A. 섹션별 CTA
- 각 데이터 섹션 옆에 작은 버튼 `💬 자세히 묻기`
- 클릭 → 운명상담사 페이지 + URL 파라미터로 prefill
  - `?prefill=내 격국 정관격이 어떻게 작용해?`

### B. 12 교차 카드 → 카드별 CTA
- 각 교차 카드 (정체성 / 욕망 / 사회역할 등) 클릭
- → 운명상담사로 자동 prefill: "내 정체성 (일간 乙 × Sun 처녀자리) 자세히 풀어줘"

### C. 정통 깊이 (Level 3) → "이게 뭐임?" 버튼
- 마니아 아닌데 펼친 사람을 위해
- Almuten 섹션 → "내 주재 행성이 인생에 어떻게 작용하나?"
- Lots 섹션 → "내 Fortune lot 의 위치 의미 알려줘"

### D. 직업/결혼/건강/재물 — 별도 카드 없음
- 차트 페이지에 *테마 카드 안 만듦* (가짜 score 거부 정신)
- 대신 페이지 하단 `테마 별로 물어보기` 영역
  - 빠른 prompt 6 개: 직업 / 결혼 / 건강 / 재물 / 학업 / 인간관계
  - 클릭 → 운명상담사 prefill

## 7. 구현 Phase

### Phase A — 데이터 받기 (`buildReportContext` 확장)
- §6.G implementation 시작
- `includeHellenistic:true` 옵션 켜기 (정적 부분만 — Lots / Almuten / dignity 5-tier / extraPoints 받음)
- 사주 7 분석 직접 호출 (12운성 / 신살 / 십신분포 / 공망 / 통근 / 득령 / 격국심층)
- `ReportFacts` shape 정의

### Phase B — adapter 갱신
- `natalToReportData` 가 새 facts shape 받게
- `buildCrossRows` 가 신규 데이터 (Almuten / Lots / dignity 5-tier) 활용해 12 카드 정밀화

### Phase C — UI Level 2 (친화 라벨 + 툴팁)
- 사주 4기둥 표 등 기존 Level 2 데이터에 친화 라벨 + 툴팁 적용
- 운명상담사 CTA `💬 자세히 묻기` 섹션별 추가

### Phase D — UI Level 3 (정통 깊이 접고 펼침)
- 새 컴포넌트 `<AdvancedSection>` (default collapsed)
- 안에 dignity 5-tier 표 / 7 Arabic Lots / Almuten Figuris / aspects minor
- 영문 + 한글 라벨 같이

### Phase E — 운명상담사 prefill 연계
- 운명상담사 라우트에 `?prefill=...` 파라미터 지원
- 차트 페이지의 CTA 가 prefill 로 자동 질문 보냄
- 페이지 하단 테마 prompt 6 개 영역

## 8. 검증 — UX 정직성

- ❌ **가짜 score** 안 만듦: `직업 적성 87점` 같은 라벨 0
- ❌ **테마별 카드** 안 만듦 (직업/결혼/건강 카드 자동 생성 0)
- ✅ **데이터 + 추상 의미** (12 교차 카드 까지만) 자동 노출
- ✅ **테마별 해석** = 운명상담사 LLM 으로 명시적 위임 (사용자 클릭)
