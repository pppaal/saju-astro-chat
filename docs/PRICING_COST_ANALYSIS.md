# 크레딧 가격 vs 토큰 비용 분석

> 작성일: 2026-05-26
> 결론: **평균 케이스 안전, 헤비 상담 유저 + ultimate 팩 조합만 리스크**

## TL;DR

| 영역                            | 마진       | 평가                                                    |
| ------------------------------- | ---------- | ------------------------------------------------------- |
| 타로 (1크)                      | **90%+**   | 매우 안전                                               |
| 사주/타로 후속 (1크)            | **90%+**   | 매우 안전                                               |
| 궁합 (1크, 5턴 평균)            | **50~70%** | 안전                                                    |
| 사주 상담 세션 (1크, 30분 윈도) | **15~50%** | 양호하지만 **20턴 풀로 쓰면 ultimate 팩에서 손해 가능** |

**액션 권장:**

1. ✅ 현재 가격 유지 OK
2. ⚠️ 상담 세션 `TURNS_PER_SESSION` 20 → **12~15로 축소** 검토 (꼬리 리스크 차단)
3. 📊 Prompt cache hit rate 모니터링 (80% 미만 떨어지면 즉시 알람)

## 1. 입력 데이터

### 모델 (`src/lib/llm/claude.ts:17,38`)

- **기본:** `claude-haiku-4-5-20251001` — **$1/$5 per MTok** (input/output), cache_read **$0.10/MTok**
- **프리미엄:** `claude-sonnet-4-5-20250929` — $3/$15, cache_read $0.30 (현재 거의 안 씀)
- **Prompt caching: ON** (1시간 TTL, `extended-cache-ttl-2025-04-11`) — 시스템 프롬프트 + 사주/점성 컨텍스트가 캐시됨 → 재호출 시 입력 비용 ~90% 절감

### 환율 가정

- **₩1,380 / $1** (현시점 근사) → $0.01 = ₩13.8

| 팩       | 크레딧 | 표시가  | Stripe 수수료(3.5%+₩300) | **실수령** | **실수령/크** |

## 2. 기능별 LLM 비용 (Haiku 4.5, 캐시 적용)

### A. 타로 (1크레딧, 3카드)

- 입력 ~1,200토큰 (콜드 캐시 라이트): 1,200 × $1.25/M = $0.0015
- 출력 ~2,500토큰: 2,500 × $5/M = $0.0125
- **총 비용: $0.014 ≈ ₩19**
- 마진: ultimate ₩137 - ₩19 = **₩118 (86%)** ✅

### B. 타로 (2크레딧, 10카드, 청크 A+B 병렬)

- 입력 ~2,000 (두 청크 같은 시스템 프롬프트 캐시 공유): $0.0020
- 출력 ~5,000: $0.025
- **총 비용: $0.027 ≈ ₩37 (2크에 분배 ₩19/크)**
- 마진: ultimate ₩274 (2크) - ₩37 = **₩237 (86%)** ✅

### C. 사주 / 점성 분석 (0크레딧, 무료 트라이얼)

- 입력 ~2,500 (콜드): $0.003
- 출력 ~1,500: $0.0075
- 총 ~$0.011 ≈ **₩15 (무료 제공)**
- → **유료 전환 미끼**. 7~30일 캐시되니 반복 조회는 거의 0원.

### D. 궁합 (1크레딧, 평균 5턴 대화)

- 콜드 1턴: 입력 8,000 × $1.25/M = $0.010 + 출력 3,000 × $5/M = $0.015 = $0.025
- 캐시 후속 4턴: 각 (캐시히트 입력 $0.001 + 출력 1,500 × $5/M = $0.0075) = $0.0085 × 4 = $0.034
- **총 비용: $0.059 ≈ ₩81**
- 마진: ultimate ₩137 - ₩81 = **₩56 (41%)** ✅
- 마진: plus ₩185 - ₩81 = **₩104 (56%)** ✅

### E. 사주 상담 (1크레딧 / 30분 세션, **최대 20턴**) ⚠️

- 콜드 1턴: 입력 6,000 × $1.25/M = $0.0075 + 출력 1,200 × $5/M = $0.0060 = $0.0135
- 캐시 후속 1턴: (캐시히트 $0.001 + 출력 $0.0060) = $0.0070
- **5턴 사용:** $0.0135 + 4 × $0.0070 = $0.042 ≈ ₩57
- **10턴 사용:** $0.0135 + 9 × $0.0070 = $0.076 ≈ ₩105
- **20턴 풀:** $0.0135 + 19 × $0.0070 = $0.147 ≈ **₩202** ⚠️

#### 마진표 (상담 세션)

| 턴 수  | 비용     | mini ₩307   | standard ₩221 | plus ₩185         | mega ₩158          | ultimate ₩137      |
| ------ | -------- | ----------- | ------------- | ----------------- | ------------------ | ------------------ |
| 5      | ₩57      | +₩250 (81%) | +₩164 (74%)   | +₩128 (69%)       | +₩101 (64%)        | +₩80 (58%)         |
| 10     | ₩105     | +₩202 (66%) | +₩116 (53%)   | +₩80 (43%)        | +₩53 (34%)         | +₩32 (24%)         |
| **20** | **₩202** | +₩105       | +₩19 (9%)     | **-₩17 (-9%)** ❌ | **-₩44 (-28%)** ❌ | **-₩65 (-47%)** ❌ |

→ **20턴 헤비 유저가 plus 이상 팩 쓰면 손해 발생**.

## 3. 시나리오별 손익

### 시나리오 1: 평범 유저 (plus 팩, 50크)

- 사용 분포 (추정): 타로 20크 + 사주상담 15크 + 궁합 10크 + 후속 5크
- 비용: 20×₩19 + 15×₩57 + 10×₩81 + 5×₩10 = ₩380 + ₩855 + ₩810 + ₩50 = **₩2,095**
- 수익: ₩9,253 (실수령)
- **마진: ₩7,158 (77%)** ✅

### 시나리오 2: 헤비 상담 유저 (ultimate 팩, 280크, 전부 상담 평균 10턴)

- 비용: 280 × ₩105 = ₩29,400
- 수익: ₩38,203
- **마진: ₩8,803 (23%)** ⚠️ 양호하지만 빠듯

### 시나리오 3: 헤비 상담 유저 (ultimate, 전부 상담 **20턴 풀**)

- 비용: 280 × ₩202 = ₩56,560
- 수익: ₩38,203
- **손실: -₩18,357 (-48%)** ❌

→ 시나리오 3는 흔치 않지만 가능. **꼬리 리스크 존재**.

## 4. 캐시가 망가지면?

만약 prompt cache hit rate가 0%로 떨어지면 (TTL 갱신 실패, 컨텍스트 변경 등):

- 상담 10턴: $0.0135 + 9 × (6,000×$1/M + 1,200×$5/M) = $0.0135 + 9×$0.012 = $0.122 ≈ **₩168**
- ultimate 280크 × ₩168 = **₩47,040 → ₩38,203 수익 → 손실 ₩8,837** ❌

→ 캐시는 **사업 필수 인프라**. cache_read 비율 모니터링 알람 필수.

## 5. 권장 가드레일

### 즉시 (코드 변경)

1. **상담 세션 턴 제한 축소: 20 → 12**
   파일: `src/app/api/counselor/realtime/route.ts:69` (`TURNS_PER_SESSION`)
   - 12턴 비용: $0.0135 + 11×$0.007 = $0.091 ≈ ₩125
   - ultimate에서도 ₩137 - ₩125 = **₩12 (9%) 마진 확보**
   - UX 영향: 12턴이면 충분히 깊은 대화 가능. 더 필요하면 새 세션 = 추가 1크레딧.

2. **궁합 무료 트라이얼 한도 점검** (`src/app/api/compatibility/counselor/route.ts:678`)
   - 비로그인 2턴 무료 → 어뷰징 가능. 익명 IP 차단 또는 1턴으로 축소 검토.

### 운영 (모니터링)

3. **Prompt cache hit rate 대시보드** 추가
   - 80% 미만 떨어지면 슬랙 알람
   - Anthropic API 응답의 `usage.cache_read_input_tokens` 비율로 계산

4. **유저당 월 LLM 비용 로그**
   - 한 유저가 한 주에 ₩10,000+ LLM 비용 발생하면 알람 (어뷰징 가능성)

### 장기 (가격 정책)

5. **ultimate 팩 (280크) 마진 압박** 인지
   - 가격 그대로 유지하되, "최강 효율 = 큰 팩"이라는 마케팅 메시지는 그대로 (어차피 평균 유저는 마진 좋음)
   - 단, **상담 헤비 유저에게는 별도 구독제 제안** 검토 (월 ₩X로 무제한 상담 N분)
