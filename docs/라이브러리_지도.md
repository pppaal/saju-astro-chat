## 3. "destiny\*" 이름 4종 — 정리 결과

| 옛 이름                                    | 실제 정체                           | 처리                                                                  |
| ------------------------------------------ | ----------------------------------- | --------------------------------------------------------------------- |
| `lib/destiny-map/`(calendar)               | calendar 상수/타입                  | ✅ → `lib/calendar/`                                                  |
| `lib/destiny-map/`(나머지)                 | report 코드                         | ✅ → `lib/report/`                                                    |
| `components/destiny-map/charts`            | report 차트                         | ✅ → `components/report/`                                             |
| `components/destinypal`·`types/destinypal` | calendar UI/타입                    | ✅ → `components/calendar/`·`types/calendar/`                         |
| `components/destiny-map/`(나머지)          | counselor 채팅                      | 유지(요청대로 counselor 미이동) — 이제 채팅 코드만 남음               |
| `lib/destiny/`                             | 컨텍스트 facts (counselor + report) | 유지 — 공유 facts(원하면 `lib/facts/` 로 후속 rename 가능)            |
| `app/destinypal`(라우트)                   | calendar 페이지 URL                 | 유지 — `/destinypal` canonical + `/calendar` 308 alias(URL 결정 별도) |

> "destiny" 가 facts·report·calendar·counselor 4개를 가리키던 혼란 해소. 남은 선택:
> ① `lib/destiny`→`lib/facts` ② 공개 URL `/destinypal`→`/calendar` 뒤집기(SEO/리다이렉트 영향).
