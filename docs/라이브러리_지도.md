## 3. "destiny\*" 이름 4종 — 혼란의 정체 & 목표 경계

| 현재 이름                    | 실제 정체                                        | 목표                                                                     |
| ---------------------------- | ------------------------------------------------ | ------------------------------------------------------------------------ |
| `lib/destiny/`               | 컨텍스트 facts (destiny counselor + report 공유) | `lib/facts/` (공유로 명시)                                               |
| `lib/destiny-map/`           | **report + calendar 혼재**                       | report부 → `lib/report/` / `calendar/`·`destinyCalendar.ts` → calendar로 |
| `app+components/destinypal/` | calendar UI                                      | `…/calendar/`                                                            |
| `components/destiny-map/`    | **counselor 채팅 + report 차트 혼재**            | 채팅 → `components/counselor/` / 차트 → `components/report/`             |
| `app/destiny-counselor`      | destiny counselor 페이지                         | (유지 OK)                                                                |

> 같은 "destiny" 가 facts·report·calendar·counselor 4개를 가리켜 혼란. 목표 어휘:
> **제품 = destiny-counselor / compatibility-counselor / tarot / calendar / report**,
> 코어 = saju / astrology / chart-dictionary / cross / facts. 그 외 `destiny*`·`destinypal` 폐기.

실제 이동(rename)은 import 대량 변경이라 로컬에서 한 덩어리씩 (타입체크로 검증). §4 표가 그 목표.
