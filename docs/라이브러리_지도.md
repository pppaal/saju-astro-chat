## 3. "destiny\*" 이름 4종 — 혼란의 정체

| 이름                                                     | 실제 정체                                                      | 제품                            |
| -------------------------------------------------------- | -------------------------------------------------------------- | ------------------------------- |
| `lib/destiny/`                                           | LLM **컨텍스트 facts** 빌더(sajuFacts/astroFacts)              | 공유(상담사·리포트·궁합·캘린더) |
| `lib/destiny-map/`                                       | **리포트 + 캘린더 혼재** (natalCross=리포트, calendar/=캘린더) | 둘 섞임 ⚠️                      |
| `app+components/destinypal/`                             | **캘린더 UI**                                                  | 캘린더                          |
| `app/destiny-counselor` + `components/destiny-map`(채팅) | **상담사**                                                     | 상담사                          |

→ 같은 "destiny" 단어가 _facts·리포트·캘린더·상담사_ 4개를 가리킴. 이게 "뭐가 뭔지" 모르던 핵심 원인.

## 4. 섞인 곳 & 권장 목표 경계 (정리하면 이렇게)

> 아래는 _목표 지도_. 실제 이동(rename)은 import 대량 변경이라 로컬에서 점진 수행 권장.

| 현재 (섞임)                                        | 문제                                              | 권장 목표                                           |
| -------------------------------------------------- | ------------------------------------------------- | --------------------------------------------------- |
| `lib/destiny/`                                     | 이름이 제품(리포트?)처럼 보이나 실제론 공유 facts | `lib/context/` 또는 `lib/facts/` (공유)             |
| `lib/destiny-map/calendar/` · `destinyCalendar.ts` | 캘린더 코드가 리포트 폴더 안                      | `lib/calendar-engine/` 또는 `lib/calendar/` 로 이동 |
| `lib/destiny-map/`(나머지)                         | 리포트 전용인데 모호한 이름                       | `lib/report/`                                       |
| `components/destiny-map/`(채팅)                    | 상담사 채팅이 "destiny-map"에                     | `components/counselor/`                             |
| `components/destiny-map/charts/`                   | 리포트 차트                                       | `components/report/`                                |
| `app+components/destinypal/`                       | 캘린더인데 이름이 destinypal                      | `…/calendar/` (운흐름)                              |
| `lib/chart-dictionary/`                            | 리포트만 쓰지만 캘린더도 써야 함                  | 위치는 유지, **공유 코어**로 명시                   |

**목표 어휘 통일:** 제품 = `tarot / counselor / compatibility / calendar(운흐름) / report`.
코어 = `saju / astrology / chart-dictionary / cross`. 그 외 `destiny*` 이름은 폐기.
