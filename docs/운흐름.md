## 0. 핵심 철학 — 타이밍이 전부

운흐름의 가치는 "무엇이 좋다/나쁘다"(의미)가 아니라 **"언제 그게 겹치는가"
(타이밍)** 에 있다.

- 교차는 **시간이 겹칠 때만** 의미. cross-activation 은 사주 신호와 점성 신호가
  *같은 날짜 구간(window 교집합)* 에 동시 활성일 때만 발화한다.
- **의미 매핑을 늘리는 것 ≠ 개선.** 페어를 늘리면 교차가 더 자주 떠서 "특별한
  타이밍"이 흔해진다(변별력 ↓). 외행성(천왕·해왕·명왕)을 매핑에서 *일부러 뺀*
  것도 같은 이유 — 느린 행성이라 윈도우가 길어 노이즈가 된다.
  (cf. `convergence-heavy.ts` 의 `SLOW_PLANETS`).
- 진짜 가치는 **수렴(convergence) 감지**: 여러 독립 신호가 *좁은 한 구간에 몰리는*
  순간 = "큰 날(keyDays)". 이건 이미 구현돼 있다(§4).

> 빈 곳을 보면 "채울 기회"가 아니라 **"왜 비워뒀나(노이즈 회피?)"** 부터 묻는다.

## 0.5 교차 엔진 v3 — 수학 모델 (재설계 청사진) ⭐

> **상태:** 설계 확정, 미구현. 이 섹션이 §2.1·§2.2·§7(옛 점수·교차)을 **대체**한다.
> **근거:** 딥리서치(2026-06, 소스 23 / 검증 22). 정통 타임로드 문헌 + 증거결합 통계.

### 0.5.0 왜 (현 엔진의 근본 결함)
현 `deriveScore`는 하루 **~180개 신호의 가중평균**(`50 + (avg−1.75)×16`)이다. 신호가
많을수록 평균이 중앙으로 수렴 → **대부분 날이 ≈50**(변별 죽음). 교차는
`cross-activation`이 **동시발생을 카운트** → 늘 켜진 외행성 때문에 거의 다 발화(노이즈).
증상: 큰날 다 똑같음, "신뢰 88" 도배, 점수 평평, 인트로 사주-only. (밴드·heavy필터·
신뢰도 등 그간의 땜질은 전부 이 결함의 *우회*다 — 아래 모델이 들어가면 불필요해짐.)

### 0.5.1 평결 — 티어별 스케일 정합 교차는 **옳다**
두 전통 다 *시간척도별 중첩 기법 계층*을 native로 갖는다 → 느린 것끼리·빠른 것끼리
맞추는 게 "같은 것끼리 비교"의 정답. (출처: augurine ZR/firdaria, cantian.ai 형충회합.)

| 스케일 | 서양 | 사주 |
|---|---|---|
| 인생 | 외행성 생애주기 / Firdaria(75년·9챕터) | 대운 전환 |
| 10년~ | Zodiacal Releasing **L1**(8~30년) | 대운 십신 |
| 년 | ZR **L2** + 연간 profection(1싸인/년) | 세운 |
| 월/주 | ZR **L3** + 빠른 트랜짓 | 월운 |
| 일 | (ZR L4) + 달·내행성 정확 각 | 일진 + **형충회합** |

사주의 형충회합 = "동사"(정적 명제를 점등하는 트리거), 서양 ZR/profection도 "활성 창"을
켜는 동일 논리.

### 0.5.2 핵심 수학 — 전부 `−log P`("얼마나 드문가") 하나로
**① 신호 무게 = 희소도 = `−log P(active)`**
각 신호 *타입*이 임의의 날에 켜져 있을 확률 `P`를 측정해두고 무게로 쓴다.
- 명왕성 트랜짓 `P≈0.9` → 무게 `≈0.1` (늘 켜짐 = **자동 0점**, 손으로 죽일 필요 없음)
- 일진 형충 `P≈0.1` → 무게 `≈2.3`
- 달-화성 정확각 `P≈0.02` → 무게 `≈3.9`

**② 그날의 크기 = 그날 (스케일별) 지배신호 top-k 무게 합** — *평균 아님*.
평범한 날=작음(다 흔함), 큰 날=큼(드문 게 겹침).

**③ 교차 = 사주 무거운 신호 ⊕ 점성 무거운 신호가 같은 날·같은 방향**
- 방향(valence) `V`는 두 체계 **따로** 산출: `V = Σ(polarity × −logP)` per 체계.
- `|V_saju|`·`|V_astro|` 둘 다 큼 + **같은 부호** → **합의**(증폭).
- 둘 다 큰데 **반대 부호** → **긴장**(엇갈림 플래그, valence는 중립이되 크기는 유지).
- 한쪽만 큼 → **단독 주도**.

**④ 큰 날 = 우연 기준선 초과한 날만 (look-elsewhere 보정)**
84년 전부를 스캔하면 우연 피크가 반드시 나온다 → 모든 날에 의미부여 ❌.
한 체계 타임라인을 **원형 시프트**(circular time-shift, 스트림 내부 autocorrelation 보존)로
무작위 N회 밀어 "우연이면 이 정도 겹친다"의 분포를 만들고, 실제 크기가 그 상위 분위수를
넘는 **top-k 창만** 큰 날로. (출처: LIGO time-slides arXiv 2512.17372 / look-elsewhere
Gross&Vitells.)

> 직관: 동시발생 카운트 ❌ → **"독립이라면 이만큼 겹쳤을까"로부터의 이탈**(PMI/−log-lift,
> Barlow `P(A,B) ≫ P(A)P(B)`). 두 always-on 신호는 자동으로 0 surprise.

### 0.5.3 확정 결정 (리서치 미결 → 답)
1. **P(base-rate) 측정:** 그 사람 **본인 84년 스팬**에 extractor를 돌려 신호 타입별
   활성일 비율로 측정(개인 차트의 실제 리듬 반영). 외행성→1 근처, 형충→작음.
2. **단일 생애 null:** 반복 trial 불가 → **원형 시프트** N(예 200)회로 null 분포 생성.
   스트림 통째로 회전(내부 의존성 보존), within-shuffle ❌.
3. **합의 vs 긴장:** **부호 1개 값**(`V_saju`, `V_astro` 비교) + 상태 플래그
   (합의/긴장/단독). 크기(surprise≥0)와 방향(signed)을 분리 출력.
4. **용신/조후/신살:** *새 신호·새 매핑 추가 금지.* 기존 신호의 **polarity 부호 보정자**로만
   (용신 부합 → +, 기신 → −). 교차에 새 페어를 안 늘림(노이즈 회피 철학 유지).

### 0.5.4 무엇을 두고 무엇을 갈아끼나
- ✅ **유지 (감각):** 18 사주 + 13 점성 extractor, `NatalContext`, `ActiveSignal` 타입,
  5층 UI 껍데기. (신호 *계산*은 정확함.)
- ♻️ **교체 (뇌):** `derivers/score.ts`(180평균) · `extractors/cross-activation.ts`
  (동시발생 카운트) · `derivers/convergence.ts`(점수부) → **신규 모듈 v3**:
  `base-rate 측정` → `−logP 무게` → `스케일별 지배신호` → `양방향 교차(PMI)` →
  `시프트 null + top-k`.

### 0.5.5 산출물 계약 (매 순간/매 셀)
단일 평균 점수 **금지**. 대신:
- `dominant[scale]` — 그 스케일 지배신호 1~3개 (사주/점성 각각)
- `crossState` — `합의 | 긴장 | 단독(saju) | 단독(astro) | 없음`
- `valence` — 부호 있는 방향 (합의일 때만 의미)
- `surprise` — 그 순간 이례도 (≥0, null 대비 분위수)
- `isKeyDay` — surprise가 trials 보정 임계 초과 여부

### 0.5.6 구현 순서
```
1. base-rate: extractor를 본인 스팬에 돌려 신호 타입별 P 테이블 산출
2. weight: −logP 무게 + 스케일별 top-k 지배신호 추출기 (사주/점성 분리)
3. cross: 양방향 valence 비교 → crossState + surprise
4. null: 원형 시프트 N회 → 분위수 임계 → isKeyDay
5. 검증: 실제 차트 N개 → "날마다 또렷이 다른가 / 큰날이 진짜 드문가" 통계 확인
6. swap: 어댑터가 v3 산출물 계약을 읽도록 전환, 옛 score/cross 제거
7. UI: 새 산출물에 맞춰 5층 재디자인
```

### 0.5.7 정직한 한계 (caveats)
- 이 "맞다"는 **전통을 충실·일관 구현 + 통계적으로 건전하게 결합**한다는 뜻. **경험적으로
  사건을 예측한다는 검증은 아님.**
- 점성/사주 기법 출처는 대부분 2차(계산기·교육), **통계 부분은 1차 동료심사**(arXiv/저널).
- **사주×서양 결합 선례·명시된 "스케일 정합" 원칙은 문헌에 없음** — 새 영역이나 근거는 탄탄.
- 두 체계는 같은 달력·인생을 공유해 **완전 독립 아님** → 합의를 단정하지 말고 *우연 기준선
  대비*로만 채점(0.5.2 ④).

### 2.1 교차신호 (cross-activation) — `extractors/cross-activation.ts`
> ⚠️ **동시발생 카운트 방식 → §0.5(v3, PMI/−logP)로 대체 예정.**
- 매핑 `data/saju-astro-mapping.ts`: 십신/신살 ↔ **고전 행성** 17쌍(A등급).
  예) 正官×토성, 偏財×수성, 정재×금성, 도화×금성, 역마×수성, 양인×화성,
  정인×목성/달.
- 발화: 두 신호 window 교집합에서만. polarity 합성(둘 다 +/+ 또는 −/− → 매핑
  polarity, 혼합 → 0), weight = 사주w×점성w×0.6. **외행성 제외는 의도**(§0).

### 2.1a 층별 교차 밴드 — `data/saju-astro-mapping.ts CROSS_LAYER_BAND`
행성마다 *어느 시간 층에서* 교차신호를 낼 수 있는지. 합성 신호 layer 가 밴드
밖이면 `cross-activation` 이 emit 안 함(`crossLayerAllowed`). "전부 켜기/끄기"가
아니라 **층마다 속도 맞는 행성만** 교차 → 외행성을 노이즈 없이 장주기에서 활용.

| 행성 | 속도 | 교차 허용 층 |
| --- | --- | --- |
| 달 | 매우 빠름 | 일 |
| 태양·수성·금성·화성 | 빠름 | 일·월 |
| 목성 | 중간 | 월·년·대운 |
| 토성 | 느림 | 년·대운 |
| 천왕·해왕·명왕 | 매우 느림 | **대운만** |

→ 외행성×십신 매핑(편관×명왕, 편인×해왕, 상관×천왕)을 추가하되 **대운 전용**으로
묶어, 일/월 노이즈 없이 "이 10년 × 명왕성 변형" 같은 장주기 교차를 잡는다.

### 2.2 합치도 (crossAgreement) — `derivers/crossAgreement.ts`
> ⚠️ **축 평균 변위 방식 → §0.5(v3, 우연 기준선 대비 surprise)로 대체 예정.**
사주축·점성축을 각각 `deriveScore` 로 점수화 → 변위 `sd=sajuAxis−50`, `ad=astroAxis−50`.
- `crossAgreementPercent`: 둘 다 같은 부호면 `50 + 50·min(|sd|,|ad|)/50`(50~100),
  반대면 `50 − …`(0~50), 혼합이면 50.
- `confidence`: `45 + min(heavyCount,6)·6 + (둘다 10) + (aligned 10 / opposed −8)`, 0~100.
- `crossVerified` = 둘 다 존재 AND `crossAgreementPercent ≥ 60`.

## 7. 점수 산출 (정확한 공식)

> ⚠️ **이 평균 모델은 §0.5(교차 엔진 v3)로 대체 예정.** 아래는 *현 구현*의 기록.

`derivers/score.ts`:
1. 층별 평균 polarity → 층 가중(대운1.0/세운0.85/월0.7/일0.55) 2단계 가중평균 `avg`.
2. `score = 50 + (avg − 1.75) × 16` (`normalizeAvgToScore`, `DERIVED_SCORE_BIAS=1.75`,
   `DERIVED_SCORE_SCALE=16`). bias 1.75 recenter 안 하면 평균 ~86 인플레.
3. 공명 보너스(+층 3개↑ 정렬 시 ±4), 패턴 보너스(±strength/100×12), soft-compress
   (raw>90 → 90+⌊(raw−90)/5⌋, 98 cap).
4. 등급(`grade.ts`): ≥74 최고 / ≥64 좋음 / ≥46 평범 / ≥33 조심 / <33 지키기
   (목표 분포 5/15/50/25/5%, 1095일 골든 보정).

테마 5축(`derivers/categories.ts`, `convergence.ts`): `career·money·love·health·growth`
(한국어 직업·재물·연애·건강·성장), 각 신호 themes 태그별 polarity 가중 누적, 종합
점수와 독립.
