> **단일 출처(SSOT).** 이전에 흩어져 있던 캘린더 문서를 하나로 통일한다:
> `DESTINYPAL_5TIER_SPEC` · `CALENDAR_LANGUAGE_GUIDE` · `calendar-display-design`

> 세운→월운→일진 5개 층에서 **"언제 흐름이 겹치는가(타이밍)"** 를 보여주는 엔진.

## 0. 핵심 철학 — 타이밍이 전부

운흐름의 가치는 "무엇이 좋다/나쁘다"(의미)가 아니라 **"언제 그게 겹치는가"
(타이밍)** 에 있다. 그래서:

- 교차는 **시간이 겹칠 때만** 의미가 있다. 엔진의 cross-activation 은 사주 신호와
  점성 신호가 *같은 날짜 구간(window 교집합)* 에 동시 활성일 때만 발화한다.
- **의미 매핑을 늘리는 것 ≠ 개선.** 페어를 늘리면 교차가 더 자주 떠서 "특별한
  타이밍"이 흔해진다(변별력 ↓). 외행성(천왕·해왕·명왕)을 매핑에서 *일부러 뺀*
  것도 같은 이유 — 느린 행성이라 윈도우가 수개월~수년이라 노이즈가 된다.
- 진짜 개선은 **수렴(convergence) 감지**: 여러 독립 신호가 *좁은 한 구간에 몰리는*
  순간을 잡아내는 것. = "큰 날(keyDays)". §4.

> 빈 곳을 보면 "채울 기회"가 아니라 **"왜 비워뒀나(노이즈 회피?)"** 부터 묻는다.

### 1.1 NatalContext (본명 — 고정 정체성)
**사주**(`src/lib/saju`, `src/lib/fusion/normalizer/saju.ts`): 4기둥, 일간, 오행

- `derivedScore: 0~100` (§3.1), `themeScores`(love/money/career/health/growth 각 0~100)

## 2. 신호 공통 모델

| 필드 | 의미 |
| --- | --- |
| `polarity` | −3~+3 (방향·강도) |
| `layer` | decadal/yearly/monthly/daily/hourly — 시간 스케일 |
| `weight` | 0~1 (본질 강도) |
| `themes` | 영향 영역 태그 |
| `evidence` | 원천 detail(행성·기둥·어스펙트·십신) — *왜*의 근거 |
| `window` | {start, peak, end} — 활성 구간 |

### 2.1 교차신호 (cross-activation) — `extractors/cross-activation.ts`
- 매핑 `data/saju-astro-mapping.ts`: 십신/신살 ↔ **고전 행성** 17쌍(A등급).
  예) 正官×토성(구조), 偏財×수성(상업), 정재×금성(가치), 도화×금성(매력),
  역마×수성(이동), 양인×화성(force), 정인×목성(배움)/달(돌봄).
- 발화: 두 신호 window 교집합에서만. polarity 합성(둘 다 +/+ 또는 −/− → 매핑
  polarity, 혼합 → 0), weight = 사주w×점성w×0.6.
- **외행성 제외는 의도** (§0): 느려서 노이즈.

### 2.2 합치도 (crossAgreement) — `derivers/crossAgreement.ts`
사주축·점성축을 각각 점수화 → `aligned/opposed/mixed` + 합치도 %(50중립~100일치)
+ 신뢰도. `cross-verified` = 둘 다 존재 AND 합치도 ≥ 60.

## 3. 5층 — 각 층이 데이터를 어떻게 받고 보여주나

| 층 | 어댑터 | 입력 | 핵심 표시 |
| --- | --- | --- | --- |
| 인생 | `toLifetime` | 대운 시퀀스 + ZR + 외행성 회귀 | 4 생애기 + 대운 spine + 마일스톤 |
| 대운 | `toDecade` | 현재 대운 + `decadal` 신호 + 교차 | 대운 기둥·십신 테마 + 10년 점수 + 합충/12운성 |

## 4. 타이밍 수렴 — 운흐름의 척추 (다음 방향)

캘린더는 층을 평균내 **한 숫자**로 뭉갠다. 운흐름은 층을 안 뭉개고 **"몇 개가
언제 겹치나"** 를 드러낸다. 이게 구조적 우위. 세 축:

### 4.1 큰 날 (convergence.keyDays) — 이미 일부 존재
여러 독립 신호(사주 대운·세운·월운·일진 + 점성 트랜짓·회귀·프로펙션)가 *좁은
한 구간에 몰리는* 날. display 설계의 "큰 날"이 이것. **카드의 주인공**으로.

### 4.2 다층 정렬 (multi-layer alignment)
"인생(약 火) → 대운(金 소모) → 세운(火 보충) + 목성회귀 = **2026 회복의 해**"
처럼, N개 층이 같은 방향일 때 헤드라인: **"5개 층 중 N개 정렬 ↑/↓"**.

### 4.3 윈도우 표시 (point → window)
캘린더는 "7/15 좋음"(점), 운흐름은 "7/10~20 빌드업→정점→소멸"(구간). 신호의
`window{start,peak,end}` 를 그대로 노출. 타이밍 제품의 핵심.

> 구현 원칙: **매핑을 늘리지 말고 수렴을 날카롭게.** 신호는 적고 특별하게.

## 5. 사용자 노출 언어 (라벨·카피 규칙)

> 엔진 내부 용어를 절대 그대로 노출하지 않는다. 행동 언어로 번역.

### 5.1 승인 등급 라벨 (5단계, UI·API·상세·액션플랜 일관)

| 등급 | 한국어 | English | 의미 |
| --- | --- | --- | --- |
| 0 | `실행 우선` | Execute-first | 핵심 과제를 밀 강한 날 |
| 1 | `활용 우선` | Leverage-first | 확인 한 단계 두면 좋은 날 |
| 2 | `운영 우선` | Operate-first | 루틴·정리·유지에 적합 |
| 3 | `검토 우선` | Review-first | 속도 늦추고 재확인·분할 |
| 4 | `조정 우선` | Adjust-first | 범위 축소·안정·하방 관리 |

### 5.2 승인 메타 라벨
`오늘 흐름 요약`(thesis) · `흐름`(phase) · `핵심 분야`(focus) · `신뢰`(reliability)
· `지금 할 것`(do now) · `주의할 것`(watch) · `좋은 시간`(best time) ·
`이렇게 읽은 이유`(why).

### 5.3 금지어 / 치환
- 금지: `공격` `방어` `국면` `레이어 0/1` `defensive` `matrix verdict` 등 엔진 누설어.
- 치환: `공격 52%/방어 48%` → `실행 여력 52%/안정 관리 48%`; `방어/재정렬 국면`
  → `정비 우선 흐름`; `레이어 0` → `핵심 조건`.

### 5.4 문장 규칙 & 카피 템플릿
1문장 1결론 + 1이유 + 1행동. 추상명사 2개 금지. 행동가능 동사 우선.
- 일일 thesis: `오늘은 [핵심 분야]에서 [흐름]. 그래서 [행동 1개]에 집중하세요.`
- 주의: `지금은 [위험]이 커지기 쉬워 [금지]보다 [안전]이 낫습니다.`
- 교차근거: `[분야] 해석에서 신호가 비교적 같은 방향이라 핵심 흐름은 안정적입니다.`

> 규칙: 새 엔진 필드는 (a) 승인 라벨에 매핑하거나 (b) 내부로만 두고 직접 렌더 금지.
> 카피 변경 시 우선 감사: `src/app/api/calendar/lib/{helpers,constants,presentationAdapter}.ts`,
> `src/components/calendar/{SelectedDatePanel,CalendarActionPlanView,CalendarMainView}.tsx`.

## 6. 화면 설계 원칙 & v2 마이그레이션 상태

### 6.1 "한 개념 = 한 집(주인)"
"큰 날" 하나가 5군데 중복 표시되던 문제 → 한 개념은 한 카드만 책임진다.
줌 연속체(인생 → 년 → 월 → 일+시) 상단 탭 고정, 같은 데이터를 스케일만 바꿔 본다.

### 6.2 v2/v3 하이브리드 — 통일 대상
- **v3**(구 `calculateYearlyImportantDates`, destiny-map): `score` blend, grade 0~5,
  EventCategory 7종, title/factor 키.
- **v2**(calendar-engine): `derivedScore`(polarity 가중합), themeScores 5종, signals,
  matchedPatterns, 6레이어.
- **현재 잔존 불일치 ⚠️**: 라우트가 "현재월 ±1달"만 v2 점수, **나머지 9달은 구 v3
  점수**를 displayScore 로 씀 → 다른 달 클릭 시 점수↔서사 출처가 달라 미묘한 불일치.
  어댑터(`cellsToYearlyDates`)가 12달 전부 v2로 일원화하면 해소(= 마이그레이션 핵심
  이득). 라우트 전환은 단계 2.
- 미노출 자산: `lifetimePivots`(인생 전체 층), 6레이어 전체 신호(현재 hourly만 부착).

### 6.3 어댑터 설계 미결(확정 필요)
- 교차검증 %·confidence: 신호 source 분포(사주 polarity 합 vs 점성 polarity 합 일치도)
  로 신설 권장.
- categories(5테마→7카테고리): money→wealth, growth→study, 임계 미달 시 general 폴백.
- title/recommendations: v2 `interpretation` + `matchedPatterns.headline/action` + ganji
  + 점수밴드 카운슬링 조립.
