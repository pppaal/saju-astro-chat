## 0. 핵심 철학 — 타이밍이 전부

운흐름의 가치는 "무엇이 좋다/나쁘다"(의미)가 아니라 **"언제 그게 겹치는가"
(타이밍)** 에 있다.

- 교차는 **시간이 겹칠 때만** 의미. cross-activation 은 사주 신호와 점성 신호가
  *같은 날짜 구간(window 교집합)* 에 동시 활성일 때만 발화한다.
- **의미 매핑을 늘리는 것 ≠ 개선.** 페어를 늘리면 교차가 더 자주 떠서 "특별한
  타이밍"이 흔해진다(변별력 ↓). 외행성(천왕·해왕·명왕)을 매핑에서 *일부러 뺀*
  것도 같은 이유 — 느린 행성이라 윈도우가 길어 노이즈가 된다.
  (cf. `convergence-heavy.ts` 의 `SLOW_PLANETS`).
- 진짜 가치는 **수렴(convergence) 감지**: 여러 독립 신호가 *좁은 한 구간에 몰리는*
  순간 = "큰 날(keyDays)". 이건 이미 구현돼 있다(§4).

> 빈 곳을 보면 "채울 기회"가 아니라 **"왜 비워뒀나(노이즈 회피?)"** 부터 묻는다.

### 1.1 NatalContext (본명 — 고정 정체성)
**사주**(`src/lib/saju`, `src/lib/fusion/normalizer/saju.ts`): 4기둥, 일간, 오행
분포, 일간 강약, 십신, 12운성, 4궁(년=조상/월=부모/일=배우자/시=자녀), 격국,
용신, 신살, 지장간, 통근, **대운 시퀀스**.
**점성**(`src/lib/astrology`, `src/lib/fusion/normalizer/astro.ts`): 본명 차트
(행성 sign·house), 품위, sect(주/야), Lot of Fortune·Spirit, **ZR L1/L2**, 트랜짓,
태양/달 회귀, 프로펙션, 프로그레션.

> (`calendar-engine/index.ts`). 고전 점성만 — 사주(고전)와 톤 통일.

### 1.2 CalendarCell (날짜당 1개, 365개)
- `signals: ActiveSignal[]` — 그날 활성 사주·점성·**교차** 신호
- `derivedScore: 0~100` (§3.1), `themeScores`(career/money/love/health/growth 각 0~100)

## 2. 신호 공통 모델

| 필드 | 의미 |
| --- | --- |
| `polarity` | −3~+3 (방향·강도) |
| `layer` | decadal/yearly/monthly/daily/hourly — 시간 스케일 |
| `weight` | 0~1 (본질 강도) |
| `themes` | 영향 영역 태그 |
| `evidence` | 원천 detail(행성·기둥·어스펙트·십신) — *왜*의 근거 |
| `active` | `{start, peak, end}` (ActiveWindow) — 활성 구간 (`types.ts`) |

### 2.1 교차신호 (cross-activation) — `extractors/cross-activation.ts`
- 매핑 `data/saju-astro-mapping.ts`: 십신/신살 ↔ **고전 행성** 17쌍(A등급).
  예) 正官×토성, 偏財×수성, 정재×금성, 도화×금성, 역마×수성, 양인×화성,
  정인×목성/달.
- 발화: 두 신호 window 교집합에서만. polarity 합성(둘 다 +/+ 또는 −/− → 매핑
  polarity, 혼합 → 0), weight = 사주w×점성w×0.6. **외행성 제외는 의도**(§0).

### 2.2 합치도 (crossAgreement) — `derivers/crossAgreement.ts`
사주축·점성축을 각각 `deriveScore` 로 점수화 → 변위 `sd=sajuAxis−50`, `ad=astroAxis−50`.
- `crossAgreementPercent`: 둘 다 같은 부호면 `50 + 50·min(|sd|,|ad|)/50`(50~100),
  반대면 `50 − …`(0~50), 혼합이면 50.
- `confidence`: `45 + min(heavyCount,6)·6 + (둘다 10) + (aligned 10 / opposed −8)`, 0~100.
- `crossVerified` = 둘 다 존재 AND `crossAgreementPercent ≥ 60`.

## 3. 5층 — 각 층이 데이터를 어떻게 받고 보여주나

| 층 | 어댑터 | 입력 | 핵심 표시 |
| --- | --- | --- | --- |
| 인생 | `toLifetime` | 대운 시퀀스 + ZR + 외행성 회귀 | 4 생애기 + 대운 spine + 마일스톤 |
| 대운 | `toDecade` | 현재 대운 + `decadal` 신호 + 교차 | 대운 기둥·십신 테마 + 10년 점수 + 합충/12운성 |
| 세운 | `toYear` | `yearly` 신호 + 프로펙션 + ZR | 세운 기둥 + 프로펙션 휠 + 월별 점수 |
| 월운 | `toMonth` | 해당 월 30 cells (+ 주입된 `converge`) | 월운 + 30일 격자 + 길/흉/주의일 + 수렴(큰 날) |
| 일진 | `toDay` | 해당 1 cell | 일진 + 점수 + 신호/교차 + Phase3 5종 |

- 타입: `src/types/destinypal/{lifetime,decade,year,month,day,user}.ts`
- 일진 Phase3 5종: 지장간·격국 성패·공망·응용격국(8종)·교차활성 (`toDay.ts:13-19`)
- 수렴(큰 날) 주입: `toMonth(opts.converge)` 만 읽는다(재계산 X). toYear/toDay 는
  수렴을 안 씀 — 표면화 작업 시 후보(§4).

## 4. 타이밍 수렴 — 운흐름의 척추 ⭐ (대부분 *이미 구현됨*)

캘린더는 층을 평균내 한 숫자로 뭉갠다. 운흐름은 **"몇 개가 언제 겹치나"** 를
드러내는 게 우위. **놀랍게도 핵심 로직은 이미 백엔드에 있다** — 일은 *표면화*다.

### 이미 구현된 것 (EXISTS)

| 기능 | 파일 | 무엇 |
| --- | --- | --- |
| **큰 날 (keyDays)** | `derivers/convergence.ts` (heaviness 기준 `convergence-heavy.ts`) | `deriveConvergence(cells, topN=5)` → `keyDays[]`. 각 날: `score`, `astro[]`, `saju[]`, `bothSystems`, `meaning`. 점수 = astroHeavy + sajuHeavy + (둘 다면 약한쪽×1.5). 3일 근접 dedup. |
| **다층 정렬 (5층 공명)** | `derivers/patterns.ts:195` | 패턴 `five-layer-resonance` = 사주 ≥4층 + 점성 ≥3층이 **같은 방향** 정렬 시 발화. `saju-five-layer`(5층 전부)·`astro-five-layer`(4층 중 3+)도 별도. → `cell.matchedPatterns` + 내러티브 룰. |
| **합치도/신뢰도** | `derivers/crossAgreement.ts` | §2.2. 사주·점성 방향 일치 % + confidence + crossVerified. |
| **정점 가중(exactness)** | `convergence.ts` | 트랜짓 peak 거리로 강도 감쇠(15일 윈도, peak 1.0 → 가장자리 0.3). |
| **인생 피벗** | `derivers/lifetimePivots.ts` | 점성 마일스톤(토성리턴 등) + 대운을 ±2년 내 병합 → `bothSystems` 피벗. `Interpretation.lifetimePivots` 로 **이미 노출**. |
| **신호 윈도우** | `types.ts` ActiveWindow | 모든 신호가 `{start,peak,end}` 보유. `includeEvidence:true` 시 응답에 노출. KeyEvents.window(강한 구간 1개)도 존재. |

### 아직 안 된 것 — *표면화/집계* (MISSING, 빌드 작음)

| 빠진 것 | 현재 | 필요 |
| --- | --- | --- |
| **"N개 층 정렬" 카운트** | 패턴은 boolean(5층 정렬 발동 yes/no) | `alignedLayerCount`(예: 사주 4/5, 점성 3/4) 메타 노출 → "5층 중 4개 ↑" 헤드라인 |
| **셀/수렴 레벨 윈도우** | 신호엔 window 있으나 셀·keyDays엔 점(date)만 | `keyDays[].window{start,peak,end}` 를 구성 신호에서 집계 → "7/10~20 빌드업→정점→소멸" |
| **수렴 confidence** | keyDays 에 confidence 없음 | `deriveCrossAgreement` 를 keyDays 에 적용 |
| **수렴의 전 층 노출** | `toMonth` 만 converge 읽음 | toYear/toDay/toDecade 도 해당 스케일 수렴 노출 |

> 구현 원칙: **매핑을 늘리지 말고 수렴을 날카롭게 표면화.** 신호는 적고 특별하게.

## 5. 사용자 노출 언어 (라벨·카피 규칙)

> 엔진 내부 용어를 절대 그대로 노출하지 않는다. 행동 언어로 번역.

### 5.1 라벨 — *현재 3개 체계가 공존* (통일 필요한 실제 불일치)

| 체계 | 값 | 어디 | 사용자가 봄? |
| --- | --- | --- | --- |
| **내부 5등급** | `최고/좋음/평범/조심/지키기` (0~4) | `derivers/grade.ts:9,27-31` | ✕ (내부 도메인) |
| **UI 3단계** | `좋은 날 ☀️ / 보통 🌗 / 조심할 날 🌙` | `components/calendar/scoreGrade.ts:59-89` | **○ 실제 노출** |
| **내러티브 카피** | `실행/활용/운영/검토/조정 우선`(Execute/Leverage/Operate/Review/Adjust-first) | `app/api/calendar/lib/constants.ts:81,91,101,107,117` (EN_MESSAGES category 문자열) | 부분(서사 텍스트로) |

- **주의:** 옛 언어가이드가 "승인 5라벨 = 실행우선…"이라 했지만, **실제 UI는
  3단계(좋은날/보통/조심할날)** 다. 실행우선…은 *별도 라벨이 아니라 등급별 메시지
  카피*. 통일하려면: UI 라벨 = 3단계로 확정하고, 5등급은 내부 점수밴드로만, 실행
  우선… 카피는 등급별 권고 문구로 명시.
- **임계:** 5등급 = ≥74/≥64/≥46/≥33. UI 3단계 = `luckyMin 64 / unluckyMax 45`
  (`scoreGrade.ts:45`).

### 5.2 승인 메타 라벨
`오늘 흐름 요약`(thesis) · `흐름`(phase) · `핵심 분야`(focus) · `신뢰`(reliability)
· `지금 할 것`(do now) · `주의할 것`(watch) · `좋은 시간`(best time) ·
`이렇게 읽은 이유`(why).

### 5.3 금지어 / 치환
- 금지: `공격` `방어` `국면` `레이어 0/1` `defensive` `matrix verdict` 등 엔진 누설어.
- 치환: `공격 52%/방어 48%` → `실행 여력 52%/안정 관리 48%`; `방어/재정렬 국면`
  → `정비 우선 흐름`; `레이어 0` → `핵심 조건`.

### 5.4 문장 규칙 & 카피 템플릿
1문장 1결론 + 1이유 + 1행동. 추상명사 2개 금지. 행동가능 동사 우선.
- 일일 thesis: `오늘은 [핵심 분야]에서 [흐름]. 그래서 [행동 1개]에 집중하세요.`
- 주의: `지금은 [위험]이 커지기 쉬워 [금지]보다 [안전]이 낫습니다.`
- 교차근거: `[분야] 해석에서 신호가 비교적 같은 방향이라 핵심 흐름은 안정적입니다.`

## 6. 화면 설계 원칙 & v2 마이그레이션 상태

### 6.1 "한 개념 = 한 집(주인)"
"큰 날" 하나가 5군데 중복 표시되던 문제 → 한 개념은 한 카드만 책임진다.
줌 연속체(인생 → 년 → 월 → 일+시) 상단 탭 고정, 같은 데이터를 스케일만 바꿔 본다.

### 6.2 v2/v3 — *대부분 통일됨* (옛 문서의 "9달 불일치"는 해소됨)
- **v3**(구 destiny-map blend): 레거시. **v2**(calendar-engine `derivedScore`): 현행.
- **정정:** 라우트는 이제 **12달 전체**를 v2 로 prescore + augment 한다
  (`route.ts:427-429,557`; scope=month 면 1달). 옛 "현재월 ±1달만 v2, 나머지 9달 v3"
  는 **이미 해소됨** — 코멘트에 "이전엔 ±1달만 augment했어서".
- `cellsToYearlyDates` v2-native 어댑터 **존재**, stage 2b 에서 동적 import 로 wiring
  (`adapters/cellsToYearlyDates.ts`). 남은 정리: 어댑터 산출의 교차%/categories/title
  조립 공식 확정(§6.3).

### 6.3 어댑터 설계 미결
- 교차검증 %·confidence: 신호 source 분포(사주 polarity 합 vs 점성 polarity 합 일치)
  로 신설 권장.
- categories(5테마→7카테고리): money→wealth, growth→study, 임계 미달 시 general 폴백.
- title/recommendations: v2 `interpretation` + `matchedPatterns.headline/action` + ganji
  + 점수밴드 카운슬링 조립.

## 7. 점수 산출 (정확한 공식)

`derivers/score.ts`:
1. 층별 평균 polarity → 층 가중(대운1.0/세운0.85/월0.7/일0.55) 2단계 가중평균 `avg`.
2. `score = 50 + (avg − 1.75) × 16` (`normalizeAvgToScore`, `DERIVED_SCORE_BIAS=1.75`,
   `DERIVED_SCORE_SCALE=16`). bias 1.75 recenter 안 하면 평균 ~86 인플레.
3. 공명 보너스(+층 3개↑ 정렬 시 ±4), 패턴 보너스(±strength/100×12), soft-compress
   (raw>90 → 90+⌊(raw−90)/5⌋, 98 cap).
4. 등급(`grade.ts`): ≥74 최고 / ≥64 좋음 / ≥46 평범 / ≥33 조심 / <33 지키기
   (목표 분포 5/15/50/25/5%, 1095일 골든 보정).

테마 5축(`derivers/categories.ts`, `convergence.ts`): `career·money·love·health·growth`
(한국어 직업·재물·연애·건강·성장), 각 신호 themes 태그별 polarity 가중 누적, 종합
점수와 독립.
